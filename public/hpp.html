Nuvei Hosted Payment Page (HPP / Cashier) — Integration Test Console

HPP is the simplest Nuvei flavor: your backend signs a set of parameters and redirects the customer's browser straight to Nuvei's hosted page (/ppp/purchase.do) — no session token, no client-side SDK, minimal server logic. (1) Build a checksum-signed redirect URL server-side (the secret key must never reach the browser), (2) redirect the customer there to enter payment details on Nuvei's page, (3) Nuvei redirects back to your successUrl/failureUrl/pendingUrl with signed output parameters, and (4) the same DMN webhook used by the other two flavors delivers the final authoritative status.

Step 1Build the signed redirect URL

Calls our backend, which computes the HPP checksum server-side — SHA-256(secretKey + all parameter VALUES concatenated in submission order). This is a different formula from the REST API's field-based checksum; HPP is Nuvei's older query-string/form flow.

Step 2Redirect to Nuvei's hosted page

In production this is a real browser redirect (or full-page/IFrame load) to https://ppp-test.safecharge.com/ppp/purchase.do?...&checksum=.... Against placeholder credentials this will show Nuvei's own error page rather than a real checkout — but it confirms the signed URL construction is correct.

Step 3Return / output parameters

Nuvei can only redirect a real customer's browser to a publicly reachable successUrl/failureUrl/pendingUrl — unreachable from a local dev server, same limitation as DMNs. Use the simulate buttons to generate a correctly-signed return payload and validate it through the real checksum-verification code path.

Step 4DMN (shared with the other flavors)

HPP's notify_url (built into the Step 1 request) points at the same /webhooks/nuvei/dmn endpoint the REST API and Simply Connect flows use — DMN delivery is flow-agnostic in Nuvei's architecture. Any DMN received while this page is open will appear live in the event log on the right.
