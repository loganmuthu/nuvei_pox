Nuvei Simply Connect — Integration Test Console

Simply Connect is a hybrid flow: a lightweight embedded checkout UI rendered client-side, backed by a server-side session. (1) Your backend calls /openOrder (this flavor's documented session call — not /getSessionToken, which is specific to the REST APM flow elsewhere in this console) to get a sessionToken, since /openOrder requires the merchant secret key and must never run in the browser, (2) the frontend loads checkout.js and calls checkout() with that token to render the embedded payment form, (3) the widget's onResult callback returns a client-side result which must be treated as untrusted, and (4) the backend cross-checks via /getPaymentStatus or the same shared DMN webhook.

Step 1Backend: open order & get session token

Calls /openOrder server-side — amount/currency are required here because they're baked into the checksum, same formula as the REST API's confirmed /openOrder pattern. The response's sessionToken (not the secret key) is the only thing sent to the browser.
